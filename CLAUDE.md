# Project Documentation: AI Developer Arena

## Overview
This project is a competence development arena designed to test and compare how different local LLMs (Gemma, Qwen, etc.) manage coding tasks. The current focus is a **Retro Worm Game**. All development is done locally using Ollama and Continue.

## Project Structure
The project uses a **Task > Model Family > Specific Model** hierarchy:

- `worm-game/`: Root directory for the current coding challenge.
    - `shared/`: Framework assets (e.g., `sounds.js`)—**DO NOT MODIFY**.
    - `gemma/`: Implementations generated by Gemma models.
    - `qwen/`: Implementations generated by Qwen models.
- `TASK.md`: The master requirements and plan for the AI.
- `MEMORY.md`: The iterative state tracker. Reference this to see the current progress.
- `.continue/`: IDE configuration, project-specific rules, and prompt templates.

## AI Instructions & Constraints
- **Role**: You are a local AI developer agent. Act with high autonomy but follow the project "Laws."
- **Rules Enforcement**: Strictly follow all laws defined in `.continue/rules/` regarding the Retro UI and Audio implementation.
- **Audio Constraint**: Use the shared audio library at `../../shared/sounds.js`. Call `window.playMoveSound()`, `window.playEatSound()`, and `window.playGameOverSound()`.
- **Iterative Workflow**: This is an iterative project. Do not assume the code is perfect on the first run; audit existing files in the specific model subfolders before suggesting fixes.
- **Tools**: If using Gemma, prefer explicit tool calls. Always use `@codebase` or `@MEMORY.md` to synchronize with the latest project state.

## Hardware Environments
This project is tested across two specific environments:
1. **Mac**: Apple M4 Max (16 Cores), 48GB Unified Memory.
2. **PC**: Ryzen 5, 32GB RAM, Nvidia RTX 5070 Ti (16GB).
